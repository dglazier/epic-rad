The ePIC Implementation

Unified Reconstruction

ePIC data is distributed across multiple branches (Central Tracker, Roman Pots, ZDC). ePIC-RAD stitches these sources into a single, unified event structure.

  • Central ReconstructedParticles
  • Forward Roman Pots & ZDC
  • Truth MCParticles

MC & Matching

Built-in handling for Monte Carlo truth matching across different detectors.

  • Automatic beam energy determination from MC headers.
  • Merged rec_match_id across all detectors.
  • Generation of rec_true_pid for perfect-PID studies.

Analysis Example

#include "ePICReaction.h" #include "KinematicsProcessor.h" void ProcessY4260() { // 1. Initialize ePIC Data Source rad::epic::ePICReaction reaction("events", "data/*.root"); // 2. Setup Beams & Matching (Crucial for MC) reaction.SetBeamsFromMC(0, 1); reaction.SetupMatching(); // 3. Define Analysis Candidates // Note: "rec_pid" works for Central, RP, and ZDC tracks uniformly const int Role_ScatEle = 2; reaction.SetParticleCandidates("ele", Role_ScatEle, rad::index::FilterIndices(11), {"rec_pid"}); reaction.SetParticleCandidates("p", 3, rad::index::FilterIndices(2212), {"rec_pid"}); // 4. Configure Kinematics (Standard RAD Workflow) rad::KinematicsProcessor kine(&reaction, "rec_"); kine.Creator().Sum("Jpsi", {{"ele", "pos"}}); kine.Init(); // 5. Run & Save reaction.Snapshot("output.root"); }
