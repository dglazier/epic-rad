Handling ePIC Data

How to use ePICReaction to analyze multi-detector data from podio/ROOT files.

1Initialization & Beams

Unlike generic experiments, ePIC requires specific handling of beam energies (often encoded in MC headers) and multiple data branches.

rad::epic::ePICReaction reaction("events", "data/sim_*.root"); // 1. Set Beams from MC Header (Indices: 0=Electron, 1=Ion) // This scans the file header to determine the correct crossing angle and energy. reaction.SetBeamsFromMC(0, 1); // 2. Prepare Data Structure // Merges all detectors and builds truth tables. reaction.SetupMatching();

2Unified Reconstruction

Raw ePIC data is split into separate collections. ePICReaction uses a ParticleInjector to merge them into a single "Unified" list (rec_px, rec_pid, etc.).

You do not need to know which detector a particle came from to analyze it, but you can check if needed.

Sources Merged:

  • Central: ReconstructedParticles (ID=1)
  • Roman Pots: ForwardRomanPot... (ID=2)
  • ZDC: ReconstructedFarForward... (ID=3)
// Example: Accessing detector source in analysis // This variable is automatically available after initialization reaction.Define("is_roman_pot", "rec_det_id == 2");

3MC Matching & Truth

When you call SetupMatching(), the framework generates several critical columns for efficiency and resolution studies.

Key Variables

  • rec_match_id: Index of the MCParticle that generated this track. Value is -1 if noise/fake.
  • rec_true_pid: The PDG code of the matched truth particle. Useful for "Perfect PID" studies.
Important: Matching logic differs by detector. Central tracks use ReconstructedParticleAssociations. Forward detectors (RP, ZDC) use a heuristic based on PID and Generator Status.
// Example: Perfect PID Selection // Use the Truth PID instead of the Reconstructed PID to define candidates reaction.SetParticleCandidates("ele", 2, rad::index::FilterIndices(11), {"rec_true_pid"});

4Forward Detectors

Particles like the Recoil Proton (Roman Pots) or Neutron (ZDC) are handled transparently. However, ensure your input files actually contain these branches if you are performing a "Detected Recoil" analysis.

// Selecting a proton (Recoil) // The framework searches the Unified vector. If a proton was found in the Roman Pots, // it is included here automatically. reaction.SetParticleCandidates("recoil", 3, rad::index::FilterIndices(2212), {"rec_pid"});
